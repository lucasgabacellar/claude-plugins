# Workbook Structure Reference

## Overview

The Premissas_Decarb workbook (`Premissas_Decarb.xlsm`) is a centralized assumptions database for PSR's decarbonization studies. It contains 11 visible sheets plus one hidden translation sheet.

## Sheet Map

| Sheet | Purpose | Table Name |
|---|---|---|
| HOME | Dashboard with nav cards, buttons, language toggle | (none) |
| Sources | Master source registry | `tblSources` |
| Conversions | Unit conversion factors | (auto-named) |
| Hydrogen | H₂ production and cost assumptions | (auto-named) |
| Steel | Green steelmaking assumptions | (auto-named) |
| CCUS | Carbon capture assumptions | (auto-named) |
| Carbon Market | Carbon pricing and regulation | (auto-named) |
| Biogas_Biomethane | Biogas/biomethane production | (auto-named) |
| Electric Sector | Generation and tariff assumptions | (auto-named) |
| Economic | Macroeconomic and FX assumptions | (auto-named) |
| Change Log | Auto-populated edit history | `tblChangeLog` |
| _Lang (hidden) | Translation lookup table | (none) |

## Topic Sheet Columns (8 sheets)

Each topic sheet has one Excel Table (ListObject) with 9 columns:

| Col | Header Key | Content | Editable |
|---|---|---|---|
| 1 | (ID) | Auto-generated ID (e.g., `H-001`) | NO — locked, auto-filled by VBA |
| 2 | Assumption | Name/description of the assumption | YES |
| 3 | Value | Numeric or text value | YES |
| 4 | Unit | Unit of measurement | YES |
| 5 | Source ID | Reference to Sources table (e.g., `IEA-2024-001`) | YES |
| 6 | Access Date | Date the source was accessed | YES |
| 7 | Internal | Whether this is an internal PSR assumption | YES |
| 8 | Responsible | Person who entered/validated | YES |
| 9 | Observations | Free-text notes | YES |

## Sources Table (`tblSources`)

| Col | Header | Content | Example |
|---|---|---|---|
| 1 | ID | Auto-generated `AUTHOR-YEAR-SEQ` | `IEA-2024-001` |
| 2 | Title | Full title of the publication | "World Energy Outlook 2024" |
| 3 | Author | Author or organization name | "IEA" |
| 4 | Type | Publication type (droplist) | "Report", "Article", "Database" |
| 5 | Publication Date | Date of publication | 2024-10-15 |
| 6 | Link | URL to the source | "https://..." |

Source IDs follow the format: `{FIRST_WORD_OF_AUTHOR}-{YEAR}-{SEQ:000}`.
The plugin must NEVER write to column 1 of Sources — IDs are auto-generated.

## Change Log Table (`tblChangeLog`)

Fully locked. Only VBA writes to it via `modChangeLog.LogChange`. The plugin must NEVER write to this table directly.

| Col | Content |
|---|---|
| 1 | Assumption ID |
| 2 | Assumption name |
| 3 | Sheet name |
| 4 | Old value |
| 5 | New value |
| 6 | Timestamp |
| 7 | Responsible |
| 8 | Reason |

## Language System

The workbook is bilingual (PT/EN). The active language is stored in HOME cell AJ2 (named range `Lang`). All headers switch via VBA when the toggle changes. The plugin should respect the current language when displaying information but always write data values in their original language (don't translate assumption names or values).

## Protection Model

- No password — protection prevents accidental edits, not security
- `UserInterfaceOnly:=True` allows VBA to write freely
- Every VBA write follows: `SheetUnprotect` → write → `SheetReprotect`
- The plugin uses openpyxl (Python) to write, which bypasses Excel protection entirely — but must respect the same column restrictions
